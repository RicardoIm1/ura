2. DISEÑO DE ALGORITMOS

Fundamentos, metodología profesional, diagramas de flujo y pseudocódigo con ejemplos prácticos.

2.1 Conceptos básicos

Imagina que un algoritmo es como una receta de cocina: una serie de pasos específicos que, si los sigues al pie de la letra, te darán un resultado esperado (el platillo).

Definición formal: Un algoritmo es un conjunto ordenado y finito de operaciones simples y bien definidas que permite resolver un problema o tomar una decisión.
Características esenciales de un algoritmo

Preciso

Cada instrucción clara, sin ambigüedades ni dobles sentidos.

Definido

Mismos datos de entrada → mismo resultado, siempre consistente.

Finito

Número limitado de pasos; un proceso infinito no es algoritmo.

Partes de un algoritmo

Entrada (Input)

Datos necesarios para comenzar (ej. harina, huevos).

Proceso

Operaciones, cálculos y lógica (ej. batir, hornear).

Salida (Output)

Resultado final obtenido (ej. el pastel).

2.2 Metodología de la programación

De la idea a la acción: seguir una metodología para construir software de calidad. Fases clave:

1. Análisis

Entender el problema, identificar entradas y salidas esperadas.

2. Diseño

Definir lógica con diagramas de flujo o pseudocódigo.

3. Codificación

Traducir a lenguaje de programación (Python, Java, C++).

4. Prueba y depuración

Ejecutar con datos de prueba, corregir errores (bugs).

5. Mantenimiento

Mejorar, agregar funciones, adaptar a nuevos entornos.

2.4 Seudocódigos

Paso intermedio entre lenguaje humano y lenguaje de programación. Mezcla de lenguaje natural con elementos de programación. No se ejecuta, pero facilita la codificación final.

Compacto

Menos espacio que un diagrama de flujo.

Cercano al código

Sirve para Java, Python, etc.

Independiente

Mismo pseudocódigo → distintos lenguajes.

Estructura típica de un pseudocódigo
Algoritmo NombreDelAlgoritmo
// Definición de variables
Definir variable1, variable2 Como TipoDeDato (Entero, Real, Carácter, Lógico)

// Entrada de datos
Escribir "Mensaje para el usuario"
Leer variable1

// Proceso (Cálculos y Lógica)
variable2 <- variable1 * 2

// Estructuras de control
Si (condicion) Entonces
// Instrucciones si se cumple
Sino
// Instrucciones si no se cumple
FinSi

// Salida de datos
Escribir "El resultado es: ", variable2
FinAlgoritmo
2.4.1 Algoritmos en pseudocódigo (Ejemplos Prácticos)

Aplicamos todo. Ejemplos de los algoritmos más comunes usando pseudocódigo:

Ejemplo 1: Algoritmo Secuencial (sin decisiones)

Problema: Calcular el área de un rectángulo.

Algoritmo AreaRectangulo
Definir base, altura, area Como Real

Escribir "Ingresa la base del rectángulo:"
Leer base
Escribir "Ingresa la altura del rectángulo:"
Leer altura

area <- base * altura

Escribir "El área del rectángulo es: ", area
FinAlgoritmo
Ejemplo 2: Algoritmo Condicional (decisión Si-Entonces)

Problema: Determinar si un número es par o impar.

Algoritmo ParImpar
Definir numero Como Entero

Escribir "Ingresa un número entero:"
Leer numero

// Si el residuo de la división entre 2 es 0, es par.
Si (numero MOD 2 == 0) Entonces
Escribir numero, " es un número PAR."
Sino
Escribir numero, " es un número IMPAR."
FinSi
FinAlgoritmo
Nota: MOD es el operador módulo, obtiene el residuo de una división.
Ejemplo 3: Algoritmo Cíclico (repetición Mientras)

Problema: Sumar todos los números del 1 al N (N dado por el usuario).

Algoritmo SumaHastaN
Definir n, i, suma Como Entero

Escribir "¿Hasta qué número quieres sumar?"
Leer n

i <- 1 // Inicializamos el contador
suma <- 0 // Inicializamos la suma en cero

Mientras (i <= n) Hacer
suma <- suma + i // Al valor actual de suma, le agregamos i
i <- i + 1 // Incrementamos el contador para avanzar
FinMientras

Escribir "La suma de 1 a ", n, " es: ", suma
FinAlgoritmo
Recuerda: Un buen diseño de algoritmos es la clave para programas robustos y eficientes. Domina los fundamentos, usa diagramas o pseudocódigo según la complejidad, y practica con ejemplos como los mostrados.
Algoritmos: precisión, definición, finitud — Base de la programación.
