2. DISEÑO DE ALGORITMOS

2.1 Conceptos básicos

Para empezar a diseñar algoritmos, debemos tener claros los cimientos. Imagina que un algoritmo es como una receta de cocina: tienes una serie de pasos específicos que, si los sigues al pie de la letra, te darán un resultado esperado (el platillo).

Definición formal: Un algoritmo es un conjunto ordenado y finito de operaciones simples y bien definidas que permite resolver un problema o tomar una decisión. Características esenciales de un algoritmo:

  • Preciso: Cada instrucción debe ser clara y no dar lugar a ambigüedades. No puede tener dobles sentidos.
  • Definido: Si se sigue un algoritmo dos veces con los mismos datos de entrada, se debe obtener el mismo resultado siempre. Es consistente.
  • Finito: Debe tener un número limitado de pasos. Si es infinito, ya no es un algoritmo, es un proceso (como el loop de un sistema operativo).

Partes de un algoritmo:

  • Entrada (Input): Los datos que necesita el algoritmo para comenzar (ej. harina, huevos, azúcar).
  • Proceso: Las operaciones y cálculos que se realizan con los datos de entrada (ej. batir, hornear).
  • Salida (Output): El resultado final obtenido (ej. el pastel).

2.2 Metodología de la programación

Aquí pasamos de la idea a la acción. No es solo sentarse a teclear código, sino seguir una metodología para construir software de calidad. La metodología más común y efectiva para resolver problemas se divide en fases:

1. Análisis del problema: Entender qué se necesita resolver.

  • Leer el problema las veces que sea necesario.
  • Identificar los datos de entrada y el resultado esperado.
  • Preguntar: ¿Qué me piden? ¿Con qué datos cuento?

2. Diseño del algoritmo: Definir cómo lo vamos a resolver.

  • Aquí es donde usamos herramientas como diagramas de flujo o pseudocódigo.
  • Se plantea la lógica y la secuencia de pasos.

3. Codificación (Implementación): Traducir el algoritmo a un lenguaje de programación específico (Python, Java, C++, etc.).

4. Prueba y depuración: Ejecutar el programa para verificar que funciona.

  • Prueba: Ejecutarlo con datos de prueba (válidos, inválidos, límites) para ver si da el resultado esperado.
  • Depuración: Corregir los errores (bugs) que aparecen.

5. Mantenimiento: Mejorar el programa, agregar nuevas funciones o adaptarlo a nuevos entornos una vez que ya está en uso.

2.3 Diagramas de flujo

Un diagrama de flujo es la representación gráfica del algoritmo. Es como ver el mapa del tesoro. Usa símbolos estándar para que cualquier programador lo entienda.

Simbología principal (estándar ANSI):

  • Óvalo o Terminal: Indica el inicio o el fin del programa.
  • Paralelogramo o Datos (Entrada/Salida): Lectura de datos (input) o impresión de resultados (output).
  • Rectángulo o Proceso: Operaciones aritméticas y asignaciones (ej. Suma = A + B).
  • Rombo o Decisión: Indica una bifurcación. Tiene una pregunta, una condición, y dos o más caminos de salida (Sí/No, Verdadero/Falso).
  • Flechas o Líneas de flujo: Indican la dirección y el orden en que se ejecutan las instrucciones.

Ventajas:

  • Visual, fácil de entender la lógica general.
  • Facilita la detección de errores lógicos (ej. un ciclo que no termina).

Desventaja:

  • Si el algoritmo es muy complejo, el diagrama se vuelve enorme y difícil de manejar.

2.4 Seudocódigos

Es el paso intermedio entre el lenguaje humano y el lenguaje de programación. Es una mezcla de lenguaje natural (español, inglés) y elementos de programación. No se ejecuta en una computadora, pero es la base perfecta para escribir el código real.

Ventajas:

Ocupa menos espacio que un diagrama de flujo.

Es más cercano al lenguaje de programación final.

Es independiente del lenguaje de programación. El mismo pseudocódigo puede servir para pasarlo a Java o a Python.

Estructura típica de un pseudocódigo:

Algoritmo NombreDelAlgoritmo // Definición de variables Definir variable1, variable2 Como TipoDeDato (Entero, Real, Carácter, Lógico) // Entrada de datos Escribir "Mensaje para el usuario" Leer variable1 // Proceso (Cálculos y Lógica) variable2 <- variable1 * 2 // Estructuras de control Si (condicion) Entonces // Instrucciones si se cumple Sino // Instrucciones si no se cumple FinSi // Salida de datos Escribir "El resultado es: ", variable2 FinAlgoritmo

2.4.1 Algoritmos en pseudocódigo (Ejemplos Prácticos)

Aquí es donde aplicamos todo. Vamos a ver ejemplos de los algoritmos más comunes usando pseudocódigo.

Ejemplo 1: Algoritmo Secuencial (sin decisiones)

Problema: Calcular el área de un rectangulo.

Algoritmo AreaRectangulo Definir base, altura, area Como Real Escribir "Ingresa la base del rectángulo:" Leer base Escribir "Ingresa la altura del rectángulo:" Leer altura area <- base * altura Escribir "El área del rectángulo es: " , area FinAlgoritmo

Ejemplo 2: Algoritmo Condicional (con decisión Si-Entonces)

Problema: Determinar si un número es par o impar.

Algoritmo ParImpar Definir numero Como Entero Escribir "Ingresa un número entero:" Leer numero // Si el residuo de la división entre 2 es 0, es par. Si (numero MOD 2 == 0) Entonces Escribir numero, " es un número PAR." Sino Escribir numero, " es un número IMPAR." FinSi FinAlgoritmo

(Nota: MOD significa módulo, es el operador que obtiene el residuo de una división).

Ejemplo 3: Algoritmo Cíclico (con repetición Mientras)

Problema: Sumar todos los números del 1 al N (donde N lo da el usuario).

Algoritmo SumaHastaN Definir n, i, suma Como Entero Escribir "¿Hasta qué número quieres sumar?" Leer n i <- 1 // Inicializamos el contador suma <- 0 // Inicializamos la suma en cero Mientras (i <= n) Hacer suma <- suma + i // Al valor actual de suma, le agregamos i i <- i + 1 // Incrementamos el contador para avanzar FinMientras Escribir "La suma de 1 a " , n, " es: " , suma FinAlgoritmo

3 INTRODUCCION A LENGUAJE C

3.1 Antecedentes y características

Creado por Dennis Ritchie entre 1969-1973 en los Laboratorios Bell. Fue desarrollado para escribir el sistema operativo UNIX. Es el papá o abuelo de lenguajes modernos como C++, Java, C#, JavaScript y Python. ¿Por qué C es tan bueno? Lenguaje de medio nivel: Tiene lo mejor de ambos mundos: Alto nivel: Estructuras de control, funciones, etc. (como los lenguajes modernos). Bajo nivel: Puede manipular directamente la memoria (punteros, direcciones de memoria), lo que lo hace ideal para sistemas operativos, drivers, y programación de hardware. Eficiente y rápido: Los programas en C son muy rápidos porque están cerca del hardware. Portable: Un programa escrito en C puede compilarse y ejecutarse en casi cualquier plataforma con cambios mínimos. Estructurado: Obliga (o al menos incentiva) a escribir código organizado mediante funciones. Base de todo: Entender C es entender cómo funciona la computadora "de verdad".

3.2 Estructura básica de un programa en C

Todo programa en C tiene una estructura definida. Es como el esqueleto sobre el que construyes todo.

// 1. Directivas del preprocesador (bibliotecas) #include <stdio.h> // Esta línea permite usar entrada/salida (printf, scanf) // 2. Función principal (MAIN) - TODO programa DEBE tener una int main() { // 3. Cuerpo del programa - Aquí va la lógica // Las instrucciones terminan con punto y coma ; printf("¡Hola, mundo perron!\n"); // \n es un salto de línea // 4. Valor de retorno return 0; // 0 significa que el programa terminó correctamente } // Las llaves {} delimitan bloques de código

Explicación de cada parte:

#include <stdio.h>: Incluye la biblioteca estándar de entrada/salida. Sin esto, no puedes usar printf ni scanf.

int main(): Es la función principal. La ejecución del programa comienza aquí. El int significa que devuelve un número entero al sistema operativo.

{ }: Delimitan el bloque de código de la función.

printf(): Función para mostrar texto en pantalla.

return 0;: Indica que el programa se ejecutó sin errores.

3.3 Tipos de datos fundamentales

En C, debes decirle a la computadora qué tipo de dato vas a guardar en cada variable. Esto es porque cada tipo ocupa un tamaño diferente en memoria.

Tipo Descripción Tamaño aprox. Rango aprox. Ejemplo
char Carácter (letra, número, símbolo) 1 byte -128 a 127 o 0 a 255 'A', '5', '$'
int Número entero 2 o 4 bytes -32,768 a 32,767 o mayor 25, -100, 0
float Número decimal (precisión simple) 4 bytes ±3.4E-38 a ±3.4E+38 3.1416, -0.5
double Número decimal (doble precisión) 8 bytes ±1.7E-308 a ±1.7E+308 3.14159265359
void Vacío (sin valor) 0 - Se usa en funciones

Modificadores de tipo: Puedes ajustar los tipos con:

  • short (entero corto)
  • long (entero largo)
  • unsigned (solo positivos, duplica el rango positivo)

Ejemplo: unsigned int edad; // Solo guarda números positivos (0 a 65,535)

3.4 Identificadores y variables

Identificadores: Son los nombres que le pones a las cosas (variables, funciones, etc.).

Reglas del juego (como perrón debes seguirlas):

  • Pueden contener letras (mayúsculas y minúsculas), dígitos y guión bajo _.
  • Deben comenzar con letra o guión bajo (NUNCA con número).
  • C es sensible a mayúsculas/minúsculas: edad, Edad y EDAD son diferentes.
  • No pueden ser palabras reservadas del lenguaje (int, return, if, etc.).

Variables: Espacios en memoria donde guardamos datos.

Declaración e inicialización:

// Declaración (solo creamos la variable) int numero; float promedio; char letra; // Declaración e inicialización (crear y dar valor) int edad = 25; float temperatura = 36.5; char inicial = 'J';

3.5 Constantes

Una constante es como una variable... ¡pero que no cambia! Su valor permanece igual durante todo el programa.

Formas de definir constantes:

Usando #define (directiva del preprocesador):

#include <stdio.h> #define PI 3.1416 // No lleva punto y coma #define IVA 0.16 int main() { float precio = 100; float precioFinal = precio + (precio * IVA); printf("Precio con IVA: %f\n", precioFinal); return 0; }

Usando const (modificador de tipo):

const int DIAS_SEMANA = 7; const float GRAVEDAD = 9.81; // DIAS_SEMANA = 8; // ¡ERROR! No se puede modificar

3.6 Operadores

Los operadores son símbolos que realizan operaciones con los datos.

3.6.1 Operadores aritméticos

Para hacer matemáticas básicas.

Operador Significado Ejemplo Resultado
+ Suma 5 + 2 7
- Resta 5 - 2 3
* Multiplicación 5 * 2 10
/ División 5 / 2 2 (si es entero) o 2.5 (si es float)
% Módulo (residuo) 5 % 2 1 (el residuo de 5/2)

¡Cuidado con la división!

int a = 5, b = 2; int resultado = a / b; // Resultado = 2 (pierde los decimales) float c = 5.0, d = 2.0; float resultado2 = c / d; // Resultado2 = 2.5

3.6.2 Operadores de asignación

Para dar valores a las variables.

Operador Significado Ejemplo Equivale a
= Asignación simple x = 5; x = 5
+= Suma y asigna x += 3; x = x + 3
-= Resta y asigna x -= 3; x = x - 3
*= Multiplica y asigna x *= 3; x = x * 3
/= Divide y asigna x /= 3; x = x / 3
%= Módulo y asigna x %= 3; x = x % 3

3.6.3 Operadores relacionales

Para comparar valores. El resultado siempre es verdadero (1) o falso (0).

Operador Significado Ejemplo (si x=5) Resultado
== Igual a x == 5 Verdadero (1)
!= Diferente de x != 3 Verdadero (1)
> Mayor que x > 3 Verdadero (1)
< Menor que x < 3 Falso (0)
>= Mayor o igual que x >= 5 Verdadero (1)
<= Menor o igual que x <=4 Falso (0)

3.6.4 Operadores lógicos

Para combinar condiciones.

Operador Significado Ejemplo Resultado
&& Y (AND) - Ambas deben ser verdad (5>3) && (2<4) Verdadero (1)
|| O (OR) - Al menos una debe ser verdad (5>3) || (2>4) Verdadero (1)
! NO (NOT) - Invierte el valor !(5>3) Falso (0)

3.6.5 Operadores de incremento y decremento

Muy útiles para contadores.

Operador Significado Ejemplo Equivale a
++ Incremento en 1 x++; o ++x; x = x + 1
-- Decremento en 1 x--; o --x; x = x - 1

Diferencia entre prefijo y sufijo:

int x = 5, y; y = x++; // Primero asigna y = 5, luego incrementa x a 6 // Resultado: y = 5, x = 6 x = 5; y = ++x; // Primero incrementa x a 6, luego asigna y = 6 // Resultado: y = 6, x = 6

3.7 Entrada y salida de datos

3.7.1 Salida con printf

Muestra datos en la pantalla.

printf("formato", variables);

Especificadores de formato: Le dicen a printf cómo mostrar cada tipo de dato.

Especificador Tipo de dato
%d o %i Entero (int)
%f Decimal (float o double)
%c Carácter (char)
%s Cadena de texto (string)
%u Entero sin signo (unsigned int)
%ld Entero largo (long)

Ejemplos:

int edad = 20; float altura = 1.75; char genero = 'M'; printf("Edad: %d años\n", edad); printf("Altura: %.2f metros\n", altura); // .2f muestra 2 decimales printf("Género: %c\n", genero); printf("Datos: %d, %.1f, %c\n", edad, altura, genero);

3.7.2 Entrada con scanf

Captura datos que el usuario escribe en el teclado.

scanf("formato", &variable);

¡MUY IMPORTANTE! El símbolo & (ampersand) antes del nombre de la variable. Significa "dirección de" y le dice a scanf dónde guardar el dato en memoria.

Ejemplo completo:

#include <stdio.h> int main() { int edad; float peso; char inicial; printf("Ingresa tu edad: "); scanf("%d", &edad); // GUARDAR & SIEMPRE printf("Ingresa tu peso: "); scanf("%f", &peso); printf("Ingresa tu inicial: "); scanf(" %c", &inicial); // El espacio antes de %c limpia el buffer printf("\n--- TUS DATOS ---\n"); printf("Edad: %d años\n", edad); printf("Peso: %.2f kg\n", peso); printf("Inicial: %c\n", inicial); return 0; }

3.8 Comentarios en C

Los comentarios son texto que el compilador ignora. Sirven para explicar tu código (y para que no maldigas a tu yo del pasado).

// Esto es un comentario de una línea /* Esto es un comentario de múltiples líneas */ int main() { // printf("Esto no se ejecuta"); // Comentario para desactivar código printf("Hola mundo"); // Esto sí se ejecuta return 0; }

Aquí juntamos todo lo aprendido:

#include <stdio.h> int main() { // Declaración de variables float num1, num2, resultado; int opcion; printf("=== CALCULADORA PERRONA ===\n"); // Entrada de datos printf("Ingresa el primer número: "); scanf("%f", &num1); printf("Ingresa el segundo número: "); scanf("%f", &num2); // Menú de opciones printf("\n¿Qué operación quieres hacer?\n"); printf("1. Suma\n"); printf("2. Resta\n"); printf("3. Multiplicación\n"); printf("4. División\n"); printf("Elige una opción (1-4): "); scanf("%d", &opcion); // Proceso (usando estructura condicional) if(opcion == 1) { resultado = num1 + num2; printf("\n✅ Resultado: %.2f + %.2f = %.2f\n", num1, num2, resultado); } else if(opcion == 2) { resultado = num1 - num2; printf("\n✅ Resultado: %.2f - %.2f = %.2f\n", num1, num2, resultado); } else if(opcion == 3) { resultado = num1 * num2; printf("\n✅ Resultado: %.2f * %.2f = %.2f\n", num1, num2, resultado); } else if(opcion == 4) { if(num2 != 0) { // Validación para no dividir entre cero resultado = num1 / num2; printf("\n✅ Resultado: %.2f / %.2f = %.2f\n", num1, num2, resultado); } else { printf("\n❌ Error: No se puede dividir entre cero\n"); } } else { printf("\n❌ Opción no válida\n"); } return 0; }
